# DNS Setup Guide

This guide covers both DNS layers required by NubMail:

1. Server DNS setup (admin-owned): records for the mail server host itself.
2. User domain DNS setup (tenant-owned): records each customer domain must publish.

## 1) Server DNS Setup (Admin)

Use this for the main mail server host (example: `mails.nubcoder.com`).

### Required records

| Host | Type | Value | Notes |
| --- | --- | --- | --- |
| `mails.nubcoder.com` | A | `140.238.100.102` | Mail host must resolve publicly |
| `nubcoder.com` | MX | `10 mails.nubcoder.com` | Required for inbound mail on root domain |
| `nubcoder.com` | TXT | `v=spf1 a:mails.nubcoder.com mx ip4:140.238.100.102 ~all` | Authorizes server for outbound mail |
| `mail._domainkey.nubcoder.com` | TXT | `v=DKIM1; k=rsa; p=<server-public-key>` | Use selector from app config (`DKIM_SELECTOR`, default `mail`) |
| `_dmarc.nubcoder.com` | TXT | `v=DMARC1; p=quarantine; rua=mailto:dmarc@nubcoder.com` | DMARC policy and aggregate report address |
| `140.238.100.102` | PTR | `mails.nubcoder.com` | Set via VPS/cloud provider reverse DNS panel |

### Verify in app

Go to `Dashboard -> Admin -> Server` and click `Re-run checks`.

## 2) User Domain DNS Setup

Use this for each user domain added in Domain Management (example: `example.com`).

### Required records per domain

| Name | Type | Value | Purpose |
| --- | --- | --- | --- |
| `@` | TXT | `nubmail-verification=<token>` | Proves domain ownership |
| `@` | MX | `10 mails.nubcoder.com` | Routes inbound mail to NubMail |
| `@` | TXT | `v=spf1 include:mails.nubcoder.com ~all` | Authorizes NubMail to send for this domain |
| `_dmarc` | TXT | `v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com` | DMARC policy for the user domain |
| `<selector>._domainkey` | TXT | `v=DKIM1; k=rsa; p=<domain-public-key>` | Domain-level DKIM key generated by NubMail |
| `autodiscover` | CNAME | `mails.nubcoder.com` | Client auto-configuration |
| `autoconfig` | CNAME | `mails.nubcoder.com` | Client auto-configuration |
| `imap` | CNAME | `mails.nubcoder.com` | IMAP endpoint alias |
| `smtp` | CNAME | `mails.nubcoder.com` | SMTP endpoint alias |

Optional CNAME records:

| Name | Type | Value |
| --- | --- | --- |
| `webmail` | CNAME | `mails.nubcoder.com` |
| `pop3` | CNAME | `mails.nubcoder.com` |

### Verify in app

1. Open `Dashboard -> Domains -> <your-domain> -> View DNS Setup`
2. Add records at your registrar/DNS provider
3. Click `Verify Records`

## 3) DNS Propagation and Troubleshooting

- DNS propagation can take from a few minutes up to 48 hours.
- TXT records may be split into chunks by providers; this is normal.
- PTR must be set by your infrastructure provider, not your registrar UI.

Useful commands:

```bash
# Sender logs
docker logs nubmail-smtp-sender --tail 50

# Mail queue
docker exec nubmail-smtp-sender postqueue -p

# Check reverse DNS
dig -x 140.238.100.102 +short
```
